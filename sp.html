I found that Earth's position was off. I compared the calculations with actual planetary position data from the almanac. I did not know that these things were recorded... For some reason the calculations for Earth's heliocentric longitude was off.

I investigated the AstroLib library, looked through the source code, and found the cause for this. It was adding an extra unit of Pi to Earth's heliocentric longitude to carry out its own calculations. Subtracting that unit of Pi on my end fixed everything. All of the planets' positions are aligned with other online applications now.

A lot of the calculations involved for calculating a planet's position is really tedious. Since there are libraries out there that already do these calculations, I went and found one that would provide all the calculations that I would need to map a planet's position and to simulate their orbits. I found a Java library called AstroLib, which provides me with everything I need. Things such as: Julian date conversions, heliocentric longitude, latitude, and distance. It automatically does all the tedious stuff involving orbital elements of a planet, so I don't have to worry about those, which is great!

After I got home, I downloaded AstroLib and created a test program to see how accurate the calculations were. I tested the calculations using the examples on the sites that I found for calculating a planet's position. I found the calculations to be accurate to within at least three decimal places.
